About me

I am an undergraduate pursuing a degree in Robotics Systems Engineering at the Singapore Institute of Technology, with a strong interest in autonomous systems. I enjoy exploring technologies and applying them to engineering challenges.

My current project involves developing a ROS1-based autonomous robot using the AgileX LIMO platform. It is tested within a custom-built arena inspiried by Changi Airport Terminal 3. The LIMO uses SLAM for real-time mapping and localisation while move_base handles global and local path planning for autonomous navigation. This setup is used to evaluate mapping accuracy, obstacle handling, and overall system performance.

This portfolio showcases my technical competency with ROS1, highlighting what makes ROS a powerful tool for robotics development. It documents my journey from foundational ROS1 works to a full project, with each section accompanied by relevant technical keywords. This is to guide both technical readers and search engines through the scope and depth of my work. This portfolio concludes by framing the project within a systems approach context— showing how this project ties back to real-world deployment scenario.

Why use ROS for this project?

• Fast and flexible stack to enable modular code development, making it easy to build, test and scale robotic systems
• Supported by a large open-source community, enabling rapid debugging and development
• Includes simulation and visualization tools like RViz and Gazebo for system testing and debugging
• Easy cross-platform compatability languages of C++ or Python, making it easy to write and deploy code across different systems

Final Configuration

Below is a summary of choices for LIMO bot setup:

• Tracked wheels.
Reason: Provides high grip to crawl over uneven surfaces like humps
• RTAB-Map for SLAM.
Reason:Enables 2D and 3D mapping as well as live camera view for visual tracking
• DWA Planner for obstacle avoidance.
Reason: Provides precise and responsive local path adjustments to navigate around static obstacles
• Dijkstra Algorithm for path planning.
Reason: Stable performance without overloading CPU

LIMO Bot Recovery Behaviour

• When encountering navigation failures or dead ends:
• Performs a backward movement of ~2 cm.
• Clears its local costmap to reset obstacle perception.
• Executes an in-place spin to reorient and attempt a new path.
• Repeats up to 3 times before aborting the goal.

System Context: Applying a Systems Approach

To conclude, this section reflects on the system context of the project by applying a systems approach, with reference to SEBoK principles. This project is approached such that it operates like a real-life scenario where learners are expected to demonstrate their competencies to future industry employers. The evaluation is carried out by university professors, who act as stakeholders representing industry standards and delivery expectations.

  • The project enables learners to demonstrate competency in two key areas:
    1. Technical competency – Implementing mapping, localization, recovery behaviours, and costmap layers.
    2. Problem-solving competency – Applying a systems approach to identify problems, analyse the system context, define requirements, and develop solutions within constraints.
  • With these competencies, the project is approached as if learners are building a solution for a supervisor or client, not just fulfilling academic requirements. The arena is constructed on the assumption that the robot may one day function in operational roles.

In this way, the robot is treated not just as a product, but as a potential service enabler. This context could translate to future use cases, such as a patrol robot navigating reliably in real indoor environments like airport terminals. By designing and testing within this scenario, learners gain practical experience that builds industry trust in their ability to deliver scalable, foundational robotic systems suitable for deployment.
